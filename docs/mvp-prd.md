# 夜拾 MVP PRD

## 1. 产品概述

`夜拾 / End of Day` 是一款以睡前复盘为核心场景的 AI 对话式记录产品。用户每天晚上通过语音或文字和 agent 聊几分钟，agent 引导用户回顾今天的经历、情绪和想法，并在对话结束后自动沉淀为一条可回看的“今夜记录”。随着使用积累，系统逐步形成对用户的长期理解。

## 2. 目标

MVP 只验证两件事：

- 用户是否愿意在睡前高频使用它
- 用户是否能明显感受到“它在帮我整理，并且越来越懂我”

## 3. 目标用户

核心用户先聚焦一类：

- 20-35 岁，有轻度自我记录习惯或自我整理需求的人
- 晚上容易想很多，但不想正经写长日记
- 对 AI 陪伴、情绪整理、个人成长类产品有接受度
- 微信使用频繁，愿意先从小程序开始尝试

## 4. 核心使用场景

- 睡前 3 到 8 分钟，快速复盘今天
- 情绪有波动，想找个安全出口说几句
- 想把今天值得记住的一件事存下来
- 想回看过去几天自己在忙什么、在想什么

## 5. 核心价值主张

- 比写日记更轻：可以直接说
- 比普通聊天更有价值：会帮你整理成记录
- 比单日记录更有意义：会逐渐形成长期记忆

## 6. MVP 范围

MVP 只做一个完整闭环：

1. 用户进入首页，点击开始今晚复盘
2. 通过语音或文字与 agent 进行 3 到 5 轮对话
3. 对话结束后自动生成一条“今夜记录”
4. 用户可以查看历史记录
5. agent 在后续对话中能有限度引用过去记录，表现出“记得你”

不做：

- 社区
- 多角色 agent
- 复杂情绪报告
- 年度总结
- 多端同步的复杂能力
- 强社交分享
- 游戏化体系

## 7. 功能需求

### 7.1 首页

目标：让用户快速进入当晚记录。

功能：

- 显示欢迎语和当日日期
- 主按钮：`开始今晚复盘`
- 次入口：`文字输入`
- 历史记录入口
- 最近一次记录摘要卡片

首页文案方向：

`说几句，夜拾帮你把今天收起来。`

### 7.2 对话复盘

目标：完成一次低门槛、有引导、有沉淀的睡前对话。

输入方式：

- 语音输入
- 文字输入
- 支持混合使用

对话机制：

- agent 主动发起开场问题
- 根据用户回答追问 2 到 4 轮
- 控制总时长，不宜过长
- 语气温和、不评判、不说教

建议首版引导框架：

- 今天最想记住的一件事是什么？
- 今天让你最有感觉的瞬间是什么？
- 现在还有什么事情挂在心上吗？
- 如果给今天留一句话，你会写什么？

结束条件：

- 用户主动结束
- 达到基础引导轮次后，agent 询问是否生成今晚记录

### 7.3 今夜记录生成

目标：把一次聊天变成真正可积累的内容。

每条记录至少包含：

- 标题：自动生成，可编辑
- 日期
- 今日摘要：1 到 3 句话
- 今日事件：1 到 3 条
- 今日情绪：1 到 3 个标签或一句描述
- 挂心的事：可为空
- 值得记住的片段：一句高亮
- 原始对话：MVP 可选保留全文或摘要版

用户操作：

- 查看
- 编辑标题
- 修改摘要内容
- 删除记录

### 7.4 历史记录

目标：让用户看到内容在积累，不是一次性聊天。

功能：

- 按天倒序查看历史记录
- 每条显示日期、标题、摘要前几行、情绪标签
- 点击进入详情页
- 支持基础搜索可后置，MVP 可先不做

### 7.5 初级长期记忆

目标：让用户感受到“它记得我”，但不把系统做复杂。

MVP 只做轻量级记忆：

- 保存最近若干条记录的摘要
- 下次复盘时 agent 可有限引用最近内容
- 例如：`你前两天也提到这件事，今天感觉有变化吗？`

注意：

- 不要过度引用，避免“监视感”
- 用户可在设置里清空历史

### 7.6 基础账号与设置

功能：

- 微信登录
- 查看隐私说明
- 删除单条记录
- 清空全部记录
- 开启/关闭历史引用

## 8. 非功能要求

- 整体流程应在 3 分钟内可完成一次轻量复盘
- 语音转文字与记录生成要尽量快，避免等待过久
- 隐私表达必须清楚
- 界面应安静、轻、适合夜间情境，但首版不强依赖深色模式

## 9. 关键体验原则

- `轻开始`：打开就能说，不先填一堆资料
- `弱任务感`：不是做题，不像问卷
- `强沉淀感`：每次结束后都能看到内容被好好整理
- `有边界的陪伴`：温和理解，但不伪装成心理咨询
- `可控隐私感`：用户知道哪些内容被存、怎么被用

## 10. MVP 页面清单

- 首页
- 对话页
- 今夜记录页
- 历史列表页
- 记录详情页
- 设置页
- 登录/授权弹层

## 11. 核心流程

### 首次使用

1. 进入小程序
2. 看到产品一句话说明
3. 微信授权登录
4. 进入首页
5. 开始第一次今晚复盘
6. 完成对话
7. 生成第一条今夜记录
8. 引导用户明晚继续

### 日常使用

1. 打开首页
2. 点击开始今晚复盘
3. 语音或文字对话
4. 生成今夜记录
5. 返回首页或查看历史

## 12. 成功指标

MVP 先看这几项：

- 首次完成复盘率
- 生成记录率
- 次日回访率
- 7 日内复盘次数
- 平均单次对话时长
- 用户是否编辑生成记录
- 用户是否查看历史记录

如果只能盯最关键的 3 个：

- `首次完成率`
- `7 日留存`
- `历史记录查看率`

## 13. 风险点

- 用户觉得像普通 AI 聊天，缺少独特价值
- 总结质量不稳定，导致“被误解”
- 语音识别和响应速度影响体验
- 隐私表达不到位导致用户不敢说真话
- 引导过强像问卷，引导过弱又缺少结构

## 14. MVP 版本定义

版本号可先定义为 `v0.1`

上线标准：

- 用户可完成一次文字复盘
- 用户可完成一次语音复盘
- 能生成并保存今夜记录
- 能查看历史记录
- agent 可引用最近历史内容
- 有基础隐私与删除能力

## 15. 一句话版本

`夜拾 v0.1` 是一个微信小程序，帮助用户在睡前通过语音或文字和 AI 聊几分钟，并自动生成可积累、可回看的每日记录。
